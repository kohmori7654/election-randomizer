## 背景・経緯

### P20-1: 連立計算バグ
`SeatSummary.tsx:19` で連立合計を計算しているが、`['ldp', 'sansei']`（自民＋参政党）になっている。
ユーザー指定の連立与党は「自民＋維新」なので `['ldp', 'ishin']` に修正が必要。

```typescript
// 現状（バグ）
const coalitionTotal = (['ldp', 'sansei'] as PartyId[]).reduce(...)
// 正しい実装
const coalitionTotal = (['ldp', 'ishin'] as PartyId[]).reduce(...)
```

### P20-2: 政党順序
現状は議席数降順でソートされているため、議席数によって自民と維新が隣り合わない場合がある。
自民(ldp)の直後に維新(ishin)を配置するカスタム順序が必要。

実装方針：
1. 議席数降順でソート後、維新(ishin)を見つけて自民(ldp)の直後に移動する
2. または事前定義の政党順序テーブルを使う

### P20-3: 3分の2ライン
衆議院の定数 465 × 2/3 ≈ 310議席。
憲法改正発議に必要な議席数（衆参それぞれ3分の2）として重要な指標。

定数: `const TWO_THIRDS = 310`

## 仕様

### P20-3 ラインの表示仕様
- 「過半数 233」ラインと同じデザイン（縦線＋ラベル）
- ラベルは「2/3 310」または「3分の2 310」
- 過半数ラインより右側に表示（310 > 233）
