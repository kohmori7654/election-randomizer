# Phase 21: 日本地図 地域間の視覚的分離

## 背景・問題

現在のグリッドレイアウトでは本州・四国・九州がセル間の 2px GAP しかなく、視覚的につながって表示される。
実際の地理（瀬戸内海・豊後水道・関門海峡）を反映した空白を各地域間に設けたい。

ユーザー要件（2026-06-16）:
> 本州と四国、九州がつながって表示されるのが気になる。各地域を離して表示して。
> マップが広がっても問題ない。

## 分離対象

| 海峡・海域 | 分離すべき地域 |
|---|---|
| 瀬戸内海 | 本州 ↔ 四国 |
| 豊後水道 | 四国 ↔ 九州 |
| 関門海峡 | 本州 ↔ 九州 |

## 方針

`prefectureGrid.ts` に `offsetX?: number` フィールドを追加し、
各都道府県のセルに視覚的 X オフセットを付与する（グリッド整数値は変えない）。

- 四国4県を右方向にシフト → 本州・九州との間隔を拡大
- 九州7県を左方向にシフト → 四国・本州との間隔を拡大
- `JapanMap.tsx` の座標計算を `col * STEP + (offsetX ?? 0)` へ変更
- `VIEW_W` も offsetX を考慮して再計算

## タスク

### P21-1: グリッドデータと地域オフセット調査

**対象ファイル**: `src/data/prefectureGrid.ts`

現在のグリッドを確認し、以下を決定する:
1. 四国4県（愛媛・香川・高知・徳島）のセル位置（col/row）
2. 九州7県（福岡・佐賀・長崎・熊本・大分・宮崎・鹿児島）のセル位置
3. 本州とどの列で隣接しているかを確認
4. `PrefectureGridItem` インターフェースに `offsetX?: number` を追加
5. 四国4県に `offsetX: +20`（右シフト）、九州7県に `offsetX: -20`（左シフト）を仮設定

> オフセット量は 20px 程度から始め、ブラウザ確認後に微調整する

### P21-2: `JapanMap.tsx` の offsetX 対応と VIEW_W 更新

**対象ファイル**: `src/components/JapanMap.tsx`

1. `PREFECTURE_GRID.map` のデストラクチャリングに `offsetX = 0` を追加
2. `const x = col * STEP` を `const x = col * STEP + offsetX` に変更
3. `VIEW_W` 計算を offsetX 考慮に変更:
   ```ts
   const VIEW_W = Math.max(...PREFECTURE_GRID.map(p =>
     p.col * STEP + (p.offsetX ?? 0) + (p.w ?? 1) * CELL + ((p.w ?? 1) - 1) * GAP
   )) + MARGIN * 2
   ```
4. `VIEW_W` が負の offsetX（九州を左シフト）で縮まらないよう `Math.max(minWidth, VIEW_W)` 等の処理も検討

### P21-3: ブラウザ確認と offsetX 微調整

1. ブラウザで本州・四国・九州の間隔を目視確認
2. 不足なら offsetX を増やす（目安: 四国 +30、九州 -30）
3. 全体レイアウトが崩れていないか確認
4. TypeScript ビルドエラーなし確認
