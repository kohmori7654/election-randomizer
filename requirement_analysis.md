# 要件分析書：衆院選ランダム配置シミュレーター

## 1. プロジェクト概要

第51回衆議院議員総選挙（2026年2月8日実施）の立候補者を全員ランダムに各小選挙区へ再配置し、「もし誰がどこで出馬していたら？」を可視化するシミュレーターアプリ。

GitHub Pages で公開する静的Webアプリ。

## 5. シミュレーションロジック

### 5-1. 全体フロー

```
1. 小選挙区候補者 1,119人 をシャッフル
2. 289選挙区に候補者を割り振り（各区の定員：元の候補者数と同数）
3. 各選挙区で候補者スコアを計算
4. 最高スコアの候補者が当選
```

### 5-2. スコア計算式

```
FinalScore_i = BaseRate_i × PartyCompetitionFactor × PositionFactor_i × DistanceFactor_i × FameFactor_i
```

#### BaseRate（基準得票率）
```
BaseRate_i = originalVoteRate_i
```
候補者本来の選挙区での得票率（0〜1）。

#### PartyCompetitionFactor（票の奪い合い係数）

同じ選挙区内で「陣営が同じ候補者」が複数いる場合、票を分割。

- **陣営定義**:
  - 与党陣営：自民党、日本維新の会
  - 野党左派陣営：中道改革連合、日本共産党、れいわ新選組
  - 野党中道陣営：国民民主党、チームみらい
  - 右派陣営：参政党（単独行動が多い）

- **計算**:
  - 同陣営候補が N 人いる場合：PartyCompetitionFactor = 1.0 / N^0.7（完全に等分はしない）
  - 単独の場合：1.0

#### PositionFactor（政治的マッチング係数）

```
match = 1 - |candidatePoliticalScore - constituencyVoterTrend|
PositionFactor = 0.7 + 0.6 × match
```
（0.7〜1.3 の範囲で制限）

#### DistanceFactor（地理的距離係数）

```
dist = 元の選挙区代表座標との距離（km）
DistanceFactor = max(0.5, 1.0 - dist / 2000)
```
（最大ペナルティ50%、2000km以上は一定）

#### FameFactor（知名度係数）

| ステータス | FameFactor |
|---|---|
| 現職 | 1.3 |
| 元職 | 1.1 |
| 新人 | 0.8 |

## 6. 「死の組」判定ロジック

### 定義

「死の組」＝元の選挙区で圧倒的強者（得票率 60%以上）が2人以上、同一選挙区に割り振られた組み合わせ。

### 判定条件

```
死の組スコア = 配置された強者候補者の originalVoteRate の合計
```

上位10〜20選挙区を「死の組」としてハイライト表示。

### 表示情報

- 選挙区名
- 競合する「元強者」候補者名・政党・元得票率
- 「本来であれば全員が圧勝していた」という対比表示

## 7. UI・画面構成

### デザイン方針

- **テーマ**: 選挙中継（NHK選挙速報）風
- **カラー**: 白ベース、政党カラーをアクセントに使用
- **フォント**: ゴシック系、視認性重視
- **レイアウト**: レスポンシブ（PC優先）

### 7-1. ヘッダー

```
┌─────────────────────────────────────────────┐
│ 🗳 衆院選ランダム配置シミュレーター          │
│ 第51回衆議院議員総選挙（2026年2月8日）      │
│                    [シミュレーション実行] ボタン │
└─────────────────────────────────────────────┘
```

### 7-2. 議席サマリーセクション

- 左：実際の結果（棒グラフ or ドーナツ）
- 右：シミュレーション結果
- 差分（±）を政党別に表示

```
与党計  実際: 352  → シミュレーション: ???  (±差分)
野党計  実際: 113  → シミュレーション: ???  (±差分)
```

### 7-3. 「死の組」ハイライトセクション

```
┌─────────────────────────────────────────────┐
│ ⚔ 死の組 TOP 10                             │
│ ─────────────────────────────────────────── │
│ 1. 東京5区 (死の組スコア: 1.82)             │
│    田中A（自民 元得票率68%） vs              │
│    鈴木B（中道改革 元得票率71%） vs          │
│    佐藤C（維新 元得票率62%）                 │
└─────────────────────────────────────────────┘
```

### 7-4. 地図ビジュアライゼーション（2段階ドリルダウン）

**レベル1: 都道府県マップ（デフォルト表示）**
- SVGベースの日本地図（47都道府県単位）
- 当選政党の最多政党カラーで塗り分け
- ホバー: ツールチップ（都道府県名・与党X席/野党Y席）
- クリック: レベル2へ遷移

**レベル2: 選挙区ドリルダウン（都道府県クリック後）**
- 選択された都道府県内の各選挙区をカード形式で表示
  - 選挙区名・当選者名・政党バッジ・元の選挙区名・元得票率
- 「← 全国へ戻る」ボタン

### 7-5. 選挙区別当選者リスト

- 289選挙区一覧表（ソート・フィルター対応）
- 列：選挙区名、当選者名、政党、元の選挙区、元得票率、配置スコア

### 7-6. 統計サマリー

- 与野党の転換選挙区数
- 最大の「波乱」選挙区（強者が敗れた区）
- 地方別の傾向

## 10. 非機能要件

- **パフォーマンス**: 1,119人のシャッフル＋スコア計算が100ms以内
- **ブラウザ**: Chrome/Firefox/Safari 最新版対応
- **アクセシビリティ**: 最低限のaria属性、コントラスト比確保
- **モバイル対応**: 最低限のレスポンシブ（PC優先でOK）
- **コスト**: 完全無料（GitHub Pages + GitHub Actions 無料枠）
