# ルール：コマンド実行権限

## ユーザーによる事前許可

ユーザーより以下の権限が明示的に付与されている。**確認ダイアログを出さずに即実行すること。**

> 「以降ファイルを削除する以外のコマンド実行時は許可を求めず実行して問題ありません」
> 「以降npm,npxを含むコマンド実行時も許可を求めず実行して問題ありません」
> 「以降curlを含むコマンド実行時も許可を求めず実行して問題ありません」
> 「以下のような削除を伴わないコマンド実行時は許可を求めず実行して問題ありません（forループ・パイプ等の複合コマンド含む）」

### 許可済みコマンド（確認不要・即実行）

| カテゴリ | 対象 |
|---|---|
| Bash 全般 | **ファイル削除（`rm` 系）以外のすべてのコマンド** |
| 複合コマンド | `for` / `while` ループ、パイプ（`\|`）、`&&` / `;` 連結、ヒアドキュメント等 |
| Python | `python3 <script>`, `pip3 install`, `-c` インラインスクリプト等すべて |
| Node.js | `npm install`, `npm run *`, `npx *`, `node *` すべて |
| curl | `curl` を使ったすべての HTTP リクエスト |
| git | 通常操作（`commit`, `add`, `log`, `diff` 等） |
| その他 | 上記以外も、ファイル削除でなければ確認不要 |

**判断基準**: コマンドにファイル削除・git 強制操作が含まれていなければ、どれだけ複雑なシェルスクリプトでも即実行する。

### 禁止コマンド（明示的指示なしは実行禁止）

| カテゴリ | 対象 |
|---|---|
| ファイル削除 | `rm`, `rm -rf`, `rmdir`, `unlink` 等 |
| 強制 push | `git push --force` / `git push -f` |
| 破壊的 git 操作 | `git reset --hard`, `git clean -f` 等 |

## 適用範囲

このルールはセッションをまたいで有効。ユーザーが明示的に撤回した場合のみ無効化する。
