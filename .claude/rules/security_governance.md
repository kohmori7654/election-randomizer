# ルール：セキュリティと依存関係の制約

## 実行コマンドのタイムアウト強制（Hangup Prevention）

ターミナルでコマンドを実行する際は、プロセスブロックによるトークン消費とハングアップを防止するため、
必ず Linux の `timeout` コマンドを使用すること。

```bash
# 良い例
timeout 60s npm test
timeout 60s pytest

# 悪い例（タイムアウトなし）
npm test
pytest
```

### 制約
- **デフォルト**: 60秒（`timeout 60s <コマンド>`）
- **延長許可条件**: ビルド・依存関係インストール等、明確に時間が必要な場合のみ

### エラーハンドリング
タイムアウト終了（終了コード124等）した場合:
1. 想定外事象とみなし即座に停止
2. ログを解析しプロセス状態を確認
3. 問題切り分け結果をユーザーに報告
4. **勝手な再試行・ごまかし禁止**

## 依存関係の制約
- インストール前に `npm audit` / `pip-audit` で脆弱性スキャンを実行
- 既知の脆弱性を持つパッケージのインストールはユーザーへ報告してから判断を仰ぐ
