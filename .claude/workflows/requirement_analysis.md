# ワークフロー：要件検証・アーキテクチャ設計（Architect 用）

## 前提
このワークフローは `architect` エージェントが実行する。
実装コードの生成・Bashコマンド実行は行わない。

---

## Step 1: 5項目チェック

以下の項目を順番に確認し、不足があれば**1項目ずつ**ユーザーに質問する。

1. **既存システムとの連携**: 関数名、データ構造、環境情報の明確化
2. **制約事項**: 技術的制約、使用不可ライブラリ、パフォーマンス要件
3. **入出力と挙動**: 正常系・異常系・エッジケースの定義
4. **検証・テスト**: 実装が正しいと判断できる客観的基準
5. **出力要件**: 出力形式（差分か、全ファイルか）の指定

---

## Step 2: 逐次確認（重要ルール）

- 不足項目がある場合、**一度に全て聞かずに1つずつ**質問する
- ユーザーから全ての回答が得られるまで:
  - 仮定に基づく実装コードの生成 → **禁止**
  - 次工程（Developer）への引き継ぎ → **禁止**

---

## Step 3: Implementation_Plan.md の生成

全ての要件がクリアになった段階で `Implementation_Plan.md` を生成する。

### テンプレート

```markdown
# Implementation Plan: [機能名]

## 確定仕様

### 1. 既存システムとの連携
[内容]

### 2. 制約事項
[内容]

### 3. 入出力と挙動
- 正常系: [内容]
- 異常系: [内容]
- エッジケース: [内容]

### 4. 検証・テスト基準
[内容]

### 5. 出力要件
[内容]

## Developer への引き継ぎ事項
[TDD開始に必要な情報を明記]
```

---

## 完了後のアクション

`Implementation_Plan.md` 生成後、以下を出力する:

```
✅ Architect フェーズ完了
→ Developer エージェントに引き継ぎます。
  `developer` エージェントを呼び出し、TDD を開始してください。
```
