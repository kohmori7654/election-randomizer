このページは 衆院選2026シミュレーター における「死の組」「全選挙区強豪区ランキング」の 計算式を解説します。開発中に発覚した票数データのバグと、その修正内容についても記録します。

① 支配度（dominanceRatio）

定義

各候補者の「元の選挙区でどれだけ強かったか」を表す数値です。 当選者（小選挙区）と落選者・比例復活当選者で計算式が異なります。

当選者（smd_win）:
　支配度 = 当選者の得票率 ÷ 次点者の得票率　（結果: >1.0）

落選者・比例復活当選者:
　支配度 = 自分の得票率 ÷ 当選者の得票率　（結果: 0〜1.0、惜敗率に相当）

例：石川3区（第51回衆院選2026）

候補者得票率当落支配度表示
にしだ 昭二（自民）51.9%小選挙区当選51.9÷46.1 = 1.131.13倍
こんどう 和也（中道改革）46.1%比例復活当選46.1÷51.9 = 0.888惜88.8%
南 しょうじ（共産）2.0%落選2.0÷51.9 = 0.039惜3.9%
なぜ得票率（originalVoteRate）を使うのか？
絶対的な票数（votes）はスクレイピングバグにより一部候補者で誤った値が入っていました。 得票率は各選挙区内の合計が ≈ 1.0 になることで整合性を確認でき、信頼性が高いため採用しています。 （詳細は ④ バグ発生の経緯）

② 強豪スコア（intensity）

強豪スコア = 上位3候補の平均支配度 × （強豪候補の人数 ÷ 3）

強豪候補の定義:
　・当選者の場合: 支配度 > 1.2 （元の選挙区で20%以上の差をつけて勝利）
　・落選者の場合: 惜敗率 > 0.8 （当選者の80%以上の票を獲得した僅差の落選）

閾値の根拠

閾値意味根拠
当選者 > 1.2倍20%以上の差で勝利実データの支配度分布: 中央値 1.64、平均 2.69。1.2は競争区の下限
落選者 > 0.8（惜敗率80%超）非常に僅差の落選一般に惜敗率80%超は「激戦」と呼ばれる

③ 死の組の選び方

STEP 1: 元の選挙区ごとに全候補者の支配度を計算
　　　　　当選者: 得票率 / 次点得票率 (>1.0)
　　　　　落選者: 自分の得票率 / 当選者得票率 (0~1.0 = 惜敗率)

STEP 2: シミュレーション後の全289区で強豪スコアを計算
　　　　　各区の上位3候補の支配度から avgDominance と strongCount を算出
　　　　　intensity = avgDominance × (strongCount / 3)

STEP 3: 全289区を intensity 降順でソート
　　　　　→ 上位10区が「死の組」TOP10
　　　　　→ 全289区分が「全選挙区強豪区ランキング」

重要なのは 「元の選挙区での強さ」が基準であり、 シミュレーション後の新しい選挙区でどう戦うかを予測するものではありません。 「実際の選挙で強かった人たちが同じ区に集まった」という偶然の過酷さを可視化しています。

④ バグ発生の経緯

バグ: 一部候補者の票数（votes）が正しい値の約1/100以下になっていた
影響: 票数ベースで当選者を判定していたため、誤った候補者が「当選者」と判定され、 支配度が異常な値（最大150倍以上）になっていた。

発覚したきっかけ

「全選挙区強豪区ランキング」に以下の異常値が表示されたことで発覚：

候補者表示された支配度（バグあり）正しい支配度
こんどう 和也（石川3区）90.1倍惜88.8%（比例復活当選）
日野 さりあ（愛知7区）120.6倍1.14倍（小選挙区当選）
梶原 みずほ（東京10区）150.4倍惜53.5%（落選）

バグの原因

スクレイピングスクリプト（scrape_candidates.py）が、同一選挙区内に 同じ名前（名字または名前）を持つ複数の候補者がいる場合に、 HTMLテーブルの行を誤って識別し、票数データを別候補者の行と混同していた。

選挙区混同の理由影響候補者
東京3区名前が「ひろたか」で同一石原ひろたか / 植木ひろたか
東京10区名字が「鈴木」で同一鈴木隼人 / 鈴木ようすけ
愛知7区名字が「鈴木」で同一鈴木じゅんじ / 鈴木こういち
愛知10区名前が「りょうへい」で同一三嶋りょうへい / 山内りょうへい
兵庫6区名前が「浩一郎」で同一いちむら浩一郎 / 谷浩一郎
茨城2区名前が「たくみ」で同一小沼たくみ / 宮内たくみ
石川3区特定パターンは不明にしだ昭二 / 南しょうじ
熊本4区異なる名前だが同パターン植田たかとし / 上田いたる

合計: 8選挙区・16候補者で votes フィールドに異常値を確認。

なぜ originalVoteRate は正しかったのか

originalVoteRate（得票率）はスクレイピング時に各候補者の行から独立して取得されており、 同名候補者間での混同が発生しなかった。 各選挙区内で候補者の originalVoteRate を合計すると ≈1.0 になることで正確性を確認済み。

修正2: アルゴリズム修正（P6-10A+D）

根本的な解決策として、支配度計算で votes を使わないよう変更。 今後 votes に別のバグがあっても支配度計算に影響しない。

変更前（バグあり）:
　票数（votes）の降順で当選者を判定 → 票数が誤っていると誤判定
　当選者支配度 = winner.votes / runnerUp.votes
　落選者支配度 = 0（固定）

変更後（修正済み）:
　elected === 'smd_win' で当選者を特定（票数不使用）
　当選者支配度 = winner.originalVoteRate / runnerUp.originalVoteRate
　落選者支配度 = loser.originalVoteRate / winner.originalVoteRate（惜敗率）
